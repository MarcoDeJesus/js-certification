Define a function

  • Encompass a group of script statements into a function block.
  • Keyword: function
  • Enclosing: The rest of the statements that form the function must be enclosed in curly brace { } characters.
  • Generic syntax:
    • function functionName(argument1, argument2, ...) 
{
    //statement(s) here
}

More definition rules

  • Function name

    Must begin with a letter, underscore or dollar sign character.

  • Parentheses

    Must follow the user-defined function name. These parentheses are used to receive arguments whether they exist or not.

  • Arguments

    Data that the function receives and manipulates within the function.

Calling a function

From one function to another, from a user event, or from a separate <script> block.

In this example, the function welcomeHome() is called using the onload event handler.


<script type="text/javascript">
//<!--
    function welcomeHome(message) 
    {
        alert(message);
    }
// -->
</script>

<body  onload="welcomeHome("Welcome to this course");">

Passing arguments to functions

Arguments, or parameters, are values passed into a function from outside the function.

These values are then used inside the function and discarded after the function has completed its statements.

function showName(firstName, lastName) 
{
    document.writeln("Your first name is:" + firstName);
    document.writeln("Your last name is:" + lastName);
}

Returning values

JavaScript functions can return values to the calling statement using the return statement.
Keyword:return

The value that is returned can be assigned to a variable or it can be used in an expression.

function sum(value1, value2) {
    return value1 + value2;
}

Operator precedence

Determines which expressions will be evaluated before others

Consider the expression 4 + 6 * 2, for example. JavaScript follows mathematical precedence, so multiplication takes precedence over addition, and thus the expression would evaluate to 16.

LOCAL VARIABLES

If you declare a variable within a function definition, the only way you can access or change it is from within that function.



Cannot access its value from any other function except the one in which it was declared.

GLOBAL VARIABLES

These need to be declared outside of any function definition, usually at the start of your script.



If you declare a global variable, you can then access its value from any function or script block on that page.

Anonymous function

In some cases, the function's name is omitted in the declaration. These functions are called anonymous functions.

Anonymous functions are not hoisted in JavaScript and function expressions are created at runtime, so they cannot be used before they are defined in the code.

var func = function() { 
    // function expression
}

Prototype function

Prototype property means new properties and methods can be attached.

Adding a method or property to the prototype property automatically adds that method or property to all objects of the function.

In this code block, the display() method can be accessed by any Pen object, as this method is inherited at the time of object creation

function Pen (penColor) {
    // this.penColor refers to the penColor property of the object calling the function
    this.penColor = penColor; 
 }
 
 // Add the display () method to Pen prototype property 
 Pen.prototype.display = function () {
     console.log(this.penColor);
 }
 
 var lux = new Pen ("Blue");
 
 lux.display ();     // Blue
 // in the previous statement, this points to the lux object

Closure function

Closures are inner functions that have access to the variables of the outer function - scope chain.

Closures can have one of the following three scope chains at any time: access to its own scope, access to variables in the outer function, or access to global variables.

The makeDesignation() method is defined inside the display() method, hence it has access to the variables in the display() method. A closure can be created by defining a function inside another function.

function display(name) {
    var designation = "Your name is ";
    
    fullName = function makeDesignation() {
         return designation + name;
    }
    return fullName();
}
    
display ("Alex");

Arrow function

Have the shorter syntax in comparison to conventional functions. One of the benefits of arrow functions becomes evident when the code contains various nested functions

When using an arrow function, it will retain the scope of the calling object.

function Pencil(newColor) {
    this.color = newColor; // assign the newColor argument value to the color property.
    alert(this); // alerts the current Pencil object 
    alert(this.color); // alerts yellow
    setTimeout(() => 
    {
      alert(this); // alerts the current Pencil object
      alert(this.color); // alerts yellow
    }, 1000);
}

var yellowPencil = new Pencil("yellow"); // Create a new Pencil object.

Event propagation

Event propagation takes place when a single event can be handled by more than one event handler defined at different levels of the DOM hierarchy.

  • Event bubbling

  • Event capturing

Event Bubbling

Determines which expressions will be evaluated before others

The event propagates from one handler to another starting from the handler defined for the innermost element. From there, the handlers of the parent of the target elements are triggered until the event reaches the higher-most DOM element.

Event Capturing

All the ancestor elements that were higher up in the document (to the intended target) capture the event first before passing it down.

Modern browsers usually use event bubbling unless you specify that they use event capturing.
