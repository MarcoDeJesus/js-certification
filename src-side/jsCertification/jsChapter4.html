What is Debugging?


The term debugging means troubleshooting and repairing code that does not work properly.

Why Debbug?


No developer writes perfectly clean code all the time. And sometimes, despite best efforts, code just does not perform as it should, or at all.

Types of Bugs

A bug is an erro in the code

  • Load-time errors

    Compiled or interpreted errors, usually a syntax error. Most of the time they cause error alerts.

  • Run-time errors

    Error that occurs after the script has loaded and running, usually a an improper use of commands. Generaly they cause error alerts.

  • Logic errors

    Error in proper command usage, or mathematical or structure errors; causing the script to run improperly. These error doesn't cause error alerts. Return unexpected results or failed execution.

Debugging Logic Errors


As a developer, if you take the time to deploy a properly tested script, you will decrease your maintenance debugging time to a fraction of what it could be.

Watchpoints

For troubleshooting a logic error in JavaScript, a typical watchpoint is an alert() method. You can set these alerts to display the contents of variables. After the script has been debugged don't forget to remove the alert watchpoints.

Also to comment out the code block in the review is a usefull way for troubleshoot. If the error persists, comment out another code block. Continue this process until you find the code block that is generating the error. This method uses the process of elimination to identify the error in a script.
