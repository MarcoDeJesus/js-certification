Document Object Model (DOM)


JavaScript Document Object Model (DOM) are the browser objects which provide properties and methods to create sophisticated JavaScript programs, as it was designed explicitly for Webpage use.
The JavaScript object model divides objects into three general groups: browser objects, language objects and form field objects. The following figure shows a diagram of commonly used objects recognized by JavaScript.

Challenges


Neither major browser supports all the objects shown in the figure, nor do they always support the same objects in the same manner. These compatibility issues pose a serious concern to JavaScript developers.

One of the challenges in creating effective JavaScript-enabled Web applications is overcoming compatibility issues between the major browsers

Containership

Containership relates to the principle that some objects cannot be used or referenced without a reference to the parent (container) object. 


            <script type="text/javascript">
            //<!--
                function containership() 
                {
                    var checkbox = document.myForm.chkTennis;
                }
            // -->
            </script>
            
            <body>
                <form name="myForm">
                    <input type="checkbox" name="chkTennis">
                    <label for="chkTennis">
                </form>
            </body>

The window Object - Properties

The window object is the highest-level object in the JavaScript object hierarchy and is also the default object. Properties associated with the window object

Property Description
closed Returns a Boolean value indicating whether a window is open or closed
defaultStatus Specifies or returns a string value containing the default status bar text
frames Returns an array that tracks the number of frames in a window. You can target a frame in JavaScript by referencing its frame number in the frames array
Note: Frames are no longer commonly used on the Web
length Returns an integer value representing the number of frames in the parent window. Also, returns the number of elements in the window.frames array
location Specifies or returns the location object for the current window
name Specifies or returns a string value containing the name of the window or frame
opener Returns a reference to the window that called (or opened) the current window
parent Returns a string value containing the name of the parent window, if it has one
self Returns a string value containing the name of the current window. Alternative for using the name property as previously described
status Specifies or returns a string value representing the status bar text
top Returns a string value containing the name of the topmost window
window An alternative for the self or name property

The window Object - Methods

Methods associated with the window object.

Method Description
alert("message text") Creates a dialog box with message text as the message
close() Closes the current window You must write window.close() to ensure that this command is associated with a window object, and not some other JavaScript object.
confirm("message text") Creates a dialog box with message text displayed, along with OK and Cancel buttons
moveBy(x, y)
moveTo(x, y)		 ThemoveBy() method moves a window by x pixels on the x-axis and by y pixels on the y-axis.
The moveTo() method moves the top-left corner of a window to the specified position on the x-axis and y-axis.
open("URL","name", "featureList") Opens a new window, populated by URL, with the target name of name, and with the features identified in featureList
print() Provides a script method for printing the contents of the current window and frame
prompt("message text","default text") Pops up a message box displaying message text with a text box for the user's response, which contains default text if not left empty. In some browsers, the word undefined displays in the text box if left empty.
resizeBy(x, y)
resizeTo(outerHeight, outerWidth)		 The resizeBy() method resizes a window by the specified x and y offsets along the x-axis and y-axis.
The resizeTo() method resizes a window to the height and width specified by outerHeight and outerWidth.

Event Handler Description

window object's open() method

  • open():You can launch a new window displaying an existing Webpage, or you can launch an empty window and then populate it dynamically.
  • Syntax:open("URL","WindowName","Feature List")

window attributes accesible in the open() method

The feature list consists of a series of attributes that you specify to either be displayed or not. The accepted values for each attribute are 1 or yes if the attribute is to be displayed, and 0 or no if it is not.

Attribute Description
toolbar Creates the standard toolbar, IE and Firefox only
location Creates the Location entry field, Opera only
status Creates the status bar
menubar Creates the menu at the top of the window
scrollbars Creates scroll bars when the document grows beyond the current window, IE, Firefox, and Opera only
resizable Enables resizing of the window by the user, IE only
width Specifies the window width in pixels
height Specifies the window height in pixels
top Specifies the top y coordinate onscreen where the window will open. Available in Microsoft Edge and Mozilla Firefox 1.x (and later); older browsers do not recognize it
left Specifies the left x coordinate onscreen where the window will open. Available in Microsoft Edge and Mozilla Firefox 1.x (and later); older browsers do not recognize it
fullscreen Specifies whether the browser is to be displayed in full-screen mode or not

Dynamic window

In some cases, you may need to open a new window and supply its content dynamically. In such cases, you must acquire a reference to the new window. The following example demonstrates the use of the open() method for such situations:

var newWindow;
newWindow = open("","dynamicWindow","toolbar=1,location=1");
newWindow.document.open();
newWindow.document.write("
Some text for the new window.
");
newWindow.document.close();

Dot notation revisited

  • Dot notation: dot notation is used to associate an object's name with the object's properties or methods. Some objects are themselves properties of a higher object. Dot notation is also used to depict this hierarchy 
    • objectName.objectProperty;
objectName.objectProperty = value;

objectName.objectMethod();
objectName.objectMethod(argument1,argument2,...);

parentObjectName.objectName.objectProperty;

The document object

  • The document object is subordinate to the window object in the JavaScript hierarchy.
  • It provides the properties and methods to work with many aspects of the current document, including text, forms, links, page title, and current location and URL

The document object - Properties

Following tables describes commonly used properties associated with the document object.

Property Description
anchors Returns an array containing a list of all anchors in the document
body Returns the body element of the document
cookie Returns the cookie of the document
doctype Returns the document's doctype
documentElement Returns the <html> element in the document
forms Returns an array containing references to each form in the document. A form object can also be referenced by name, via the name attribute of the <form> tag.
Note that form elements are contained within the form object.
images Returns an array containing references to all <img> tags in the document
lastModified Returns a string value representing the date and time that the document was last changed
Note: The appearance of this string is inconsistent among browsers. Also, the availability of this property may be dependent upon server configuration.
referrer Returns a string value representing the URL of the document from which the current document was accessed. Note that document.referrer only returns a value if the user arrives at the current page via an HTTP link
scripts Returns all the <script> elements in the document
title Specifies or returns a string value representing the text between the <title> and </title> tags

The document object - Methods

Following tables describes commonly used methods associated with the document object.

Method Description
close() Closes a write stream
createElement(name) Creates the HTML element specified by name
getElementById(id) Returns a reference to the element specified by its id
getElementByClassName Returns a list of elements with the given class name
getElementByTagName(name) Returns a live list of elements with the given name (live list means that the list keeps updating itself with the DOM tree automatically)
open() Opens the document to receive data from a write() stream
write(content) Writes the value of content to the document
writeln(content) Writes the value of content to the document, followed by a carriage return. Unless this command occurs within a <pre> block of preformatted text, you will see no difference between this method and the write() method.

The image object

  • The image object resides beneath the document object in the JavaScript hierarchy.
  • This object allows you to manipulate images in recent versions of all the major browsers.
  • Images can be accesed through array index and changed by inline scripting by array number or by name. 
    document.images[0].src = newImage.jpg; // images array standard syntax
<input  type="button" value="Change image" onclick="document.images[0].src='/images/myImage.gif';" /> // Inline scripting by array number
<img src="myPicture.gif" name="myImage" />
<input type="button" value="Change image"
onclick="document.images['myImage'].src='newImage.gif';" /> // Inline scripting by name

The image object - Properties

Following tables describes commonly used properties associated with the image object. The image object has no built-in methods.

Property Description
src Specifies or returns the URL of the image file. This parameter is the only one required. Following is an example of this format:
document.images[0].src = "http://www.yoursite.com/images/comp1.gif";
height and width Specifies or returns the height and width of the image in pixels. Although this parameter is not required, you should use it at all times.
length Returns the number of images for a document. You can access this property using the following syntax:
document.images.length;
complete Returns a Boolean value indicating whether the image has loaded completely.
naturalHeight and naturalWidth Returns the original height and width of an image

The image object - Event handlers

Following tables lists commonly used event handlers of the image object. The image object has no built-in methods.

Event Handler Description
onabort Processes if the user selects the Stop button while an image is loading (IE/Edge only)
onerror Processes if an error occurs while an image is loading
onload Processes either when an image's lowsrc image finishes loading, when an image's src finishes loading, or when each frame of an animated GIF appears

Handling image object events
In many applications, the <a> tag is used in conjunction with images to handle events. The <a> tag supports such event handlers as onclick, onmouseout and onmouseover.

The history object

  • The history object is subordinate to the window object in the JavaScript hierarchy.
  • You have already been using the same information used by the history object when you click the Back or Forward buttons on your browser. The browser maintains a list of recently visited Webpage URLs.
  • To access the previous pages visited, you can use the history object with the back() and forward() methods. The syntax is as follows: 
    <form>
    <input type="button" value="Back" onclick = "history.back();" />
                                            
    <input type="button" value="Forward"
    onclick = "history.forward();" />
</form>

The history object - Properties and methods

The history object has only one property associated with it, and five methods.

Property Description
length Returns an integer value representing the number of links currently referenced by the history object
Method Description
back() Sends the user to the previous page in the history list
forward() Sends the user to the subsequent page in the history list
go(x) If x is a negative integer, the user will be sent back x number of pages in the history list. If x is a positive integer, the user will be sent forward x number of pages in the history list. If x is a string value, the user will be sent to the page in the history list with a URL matching x.
pushState() Creates a new history entry
replaceState() Modifies an existing history entry

The location object

  • The location object is subordinate to the window object in the JavaScript hierarchy.
  • The location object allows you to specify URLs in a script.
  • The key property of the location object you will frequently use is the href property, which specifies or returns the hypertext reference for the current or desired URL.
  •             <form>
                <input type="button" value="Visit CIW"
                onclick="location.href = 'http://www.CIWcertified.com';" />
            </form>

            protocol://hostname:port/pathname/search#hash // URL Protocol

The location object - Properties

The location object has the following methods.

Property Description
href Specifies or returns the partial or full URL of a file or site. For example: http://www.cnn.com or page2.htm or page2.htm#Section2
protocol Specifies or returns HTTP, FTP or the protocol currently in use

The navigator object

  • Reflects information about the browser being used. This object can determine the brand and version of the browser in use, and even identify the user's operating system. 
  • var userBrowser = navigator.appName;

The navigator object - Properties and methods

The navigator object has seven properties associated with it, and only one method.

Property Description
appCodeName Returns a string value containing the code name for the client. This property returns the default Mozilla for the Mozilla browsers and also for Microsoft Edge and Google Chrome.
appName Returns a string value containing the name of the client (i.e., Netscape for the Mozilla browsers and Microsoft Internet Explorer for the Internet Explorer browsers)
appVersion Returns a string value containing the version information for the client
mimeTypes Returns an array representing all the MIME types supported by the client
platform Returns a string that identifies the operating system for which the browser was compiled
plugins Returns an array representing all the plug-ins installed on a particular browser
userAgent Returns a string value containing the complete value of the user-agent header sent in the HTTP request
Method Description
javaEnabled() Returns true if Java is enabled by the client; otherwise returns false

Virtual DOM

A virtual DOM object is a DOM object with limited capabilities. Virtual DOM objects are lightweight copies of real DOM objects that are unable to make changes to the page itself.

  • Advantages and disavantages of Virtual DOM: Every time a DOM changes the layout, this requires a lot of work from the browser. It must re-read the CSS, change the layout, and rebuild the page. With the virtual DOM there is no longer a need to update each element when the layout of the DOM changes. Also, rendering the virtual DOM is a lot faster than rendering actual browser DOM. React, a popular JavaScript library, uses the virtual DOM, making pages created with React much more efficient than pages created without the virtual DOM.

  • Common implementation of virtual DOM: Maquette, marko and cito.js are some of the many JavaScript frameworks that use virtual DOM libraries at their core. Maquette enables users to control the UI using plain JavaScript, making it easy to learn and debug.
