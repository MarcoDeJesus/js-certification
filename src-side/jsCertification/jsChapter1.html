JavaScript Characteristics

A closer look at the characteristics and features of this language.

  • Object Oriented


    1. JavaScript is an object-oriented language because it derives functionality from a collection of built-in objects. Unlike many object-oriented languages (like C or Java, which are class-based), JavaScript is prototype-based. Prototype-based programming objects are mutable, so the behavior of new instances of an object can be redefined. With JavaScript, you can also create your own custom objects.

  • Event Driven


    1. The World Wide Web is based upon an event-driven model. Events can trigger functions. Event triggers can be as simple as the user clicking a button, clicking or moving the mouse over a hyperlink, or entering text into a text field. Scripting can be tied to any of these events.
      Client-side JavaScript is part of the text within your HTML document. When your browser retrieves a scripted page, it executes the JavaScript programs and performs the appropriate operations in response to user events.

  • Platform Independent


    1. As JavaScript programs are designed to run within HTML documents, they are not tied to any specific hardware platform or operating system. However, JavaScript programs are tied to a specific user agent. Generally, these user agents are browsers. Theoretically, you can implement the same JavaScript program on any current user agent, such as Google Chrome, Mozilla Firefox, Microsoft Internet Explorer, Edge or Safari (for Apple).

  • Client-side scripts


      1. Server-side scripting is used at the backend, where the source code is not viewable or hidden at the client side (browser). On the other hand, client-side scripting is used at the front end which users can see from the browser.

      2. As against, client-side scripting does not need any server interaction.

      3. JavaScript is interpreted at runtime as opposed to being compiled so it's code can be developed and debugged easily.

      4. The client-side script can effectively minimize the load to the server.

      5. Server-side scripting is more secure than client-side scripting as the server

  • Easy to learn


    1. JavaScript is relatively easy to learn because the basic syntax is simple, you can debug it quickly, and literally, thousands of examples with comments are available for you to look-up and study all accross the web.

  • Case-Sensitive


    1. This means that the language keywords, variables, function names, and any other identifiers must always be typed with a consistent capitalization of letters. ...
      NOTE − Care should be taken while writing variable and function names in JavaScript

    JavaScript Flavors

    Java

    Java Features if you want to learn more....

    • It is compiled on a server before execution on a client.
    • It is object-oriented; applications and applets consist of object classes with inheritance.
    • Variable types must be declared (strong typing).
    • It specifies static binding; object references must exist at compile time.
    • Java Applets are called from HTML.

    JavaScript

    JS Features if you want to learn more...

    • It is interpreted (not compiled) by a client.
    • It is object-based; code uses built-in, extensible objects, but no classes or inheritance.
    • Variable data types are not declared (loose typing).
    • It specifies dynamic binding; object references checked at run time.
    • Code integrated with and embedded in HTML.

    ECMAS Backgroung:
    The inventor of JavaScript, announced in 1997 that the European Computer Manufacturer's Association (ECMA) had approved JavaScript as an international standard. In an attempt to standardize JavaScript and diminish the differences between JavaScript and JScript, ECMA released its own scripting-language specification. ECMA promoted this standard as "ECMAScript"

    This ECMA Standard is based on several originating technologies, the most well-known being JavaScript (Netscape) and JScript (Microsoft).

    ECMA team


    For more information about ECMA, consult its home page at ecma international. This site refers to the ECMAScript 2020, was published in June 2020. The most recent specification is the ninth edition, published in June 2018..

    Technologies based on JavaScript




    React

    React is used for building user interfaces. It uses different types of components to enable users to create Web applications in which data can change over time, without the need for reloading the page. This makes React fast. React Native, a set of libraries, announced by Facebook in 2015, provides a platform to develop native applications for iOS, Android, and UWP.

    Typically, React components are written in React-specific extension (JSX). The syntax for JSX is very different from JavaScript. Though developers can also use pure JavaScript

    jQuery

    jQuery was designed for making navigation in a document, creating animations, handling events, and developing AJAX applications much easier. It also enables developers to create plug-ins. Basically, it is a DOM (Document Object Model) manipulation library that simplifies the syntax for controlling DOM elements to a great extent. Used as a plug-in, jQuery is extensible. It encourages developers to separate HTML markup and JavaScript code by using simple syntax for adding event handlers, instead of using HTML event attributes. jQuery eliminates cross-browser incompatibilities and provides an interface that is consistent across browsers.

    Server-Side & Client-Side

    Server-side applications of JavaScript

    • Server-Side JavaScript (SSJS) is a JavaScript's server-side scripting environment. It allows connection to databases and supports many of the rich features and functions of any server-side scripting language. For example, it enables you to connect Webpages to databases, enables server-side image maps and saves client state so the computer will remember where the client is during a multi-page process.

    • Node.js is a JavaScript runtime environment that provides JavaScript the capability to be used for server-side scripting. It facilitates the creation of Web servers using modules. Applications developed using Node.js can run on Windows, Linux, Mac and many other servers. It enables event-driven programming on Web servers using JavaScript.
