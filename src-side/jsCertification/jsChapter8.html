Object in JavaScript


An object in Javascript is an "unordered collection of properties each of which contains a primitive value, object, or function. "Each property or method is identified by a name and mapped to a value. Javascript objects are like hash tables: a grouping of name-value pairs where the value may be data or a function.

Custom Object


It is a user-defined object that has properties and functionalities as defined by the user.

Custom JavaScript objects


Simulate database objects, but do not replace them. Custom JavaScript objects are effective when used with data that does not change often. They also provide a convenient alternative when the overhead of a database server or database application is not wanted or needed for an application. When used properly, JavaScript custom objects provide database-type functionality without a formal database server or database application.

Adventages:


  • You can create sophisticated solutions with minimal coding
  • An appropriate abstraction can be very helpful to reuse code
  • You can represent programming constructs as objects, allowing you to write efficient code schemes for evaluating forms or creating client-side databases.
  • Using appropriate data structures can help us saving memory, don’t letting all the work to the server-side application using custom objects in the client-side

    • The Constructort


    You define, or create, a custom JavaScript object with a special function called a constructor. The concept of a constructor in JavaScript is similar to that used in C++ and Java. The constructor defines the properties and methods of your object. Defining a constructor is the first step in creating an object.


    Specifically, a constructor creates an empty template from which real-time objects, called instances, can be generated. The generic syntax for creating a constructor function is as follows: 

    // Constructor
 
function NewObject(parameter) {

}

// myObject is now an object of type NewObject!

var myObject=new NewObject("I am a new object!");

    With only this information, you can code your constructor function as follows: 

    function Person(personName,personLast,personAge) {
        this.name = personName;
        this.lastName = personLast;
        this.age = personAge;
}

    This code defines a new JavaScript object template. Note that the object constructor is passed three parameters. These parameters will contain the object's properties when the object is created.

    this.name = personName;

    This statement can also be written as follows:

    this.name = name;

    the two words do not need to match. However, the right side of the equation (in this case, the word name) must match the parameter that has been declared in the argument list for the function. In most cases, using the same name for the property on both sides of the equation is simplest.

    The prototype property


    The prototype property can be used to add new properties or methods to JavaScript objects. This property can also be used with custom JavaScript objects. Consider the following example. 

    Person.prototype.knowsJS = true;
for (person in persons) {
   console.log("Knows JS? ", persons[person].knowsJS);
}                      

    
// Knows JS?  true
// Knows JS?  true
// Knows JS?  true

    This knowJS property would be available in all instances of the Person Object. The value for the knowJS property could be overridden for any instance of the Person Object. Note that if any Person instances did not have the value of the knowJS property overridden, their knowJS property would automatically inherit any value that might later be assigned to prototype.knowJS 

    Person.prototype.knowsJS;
persons[1].knowsJS = false;
persons[2].knowsJS = true;

    
for (let i in persons) {
    console.log(`${persons[i].name} knows JS?`,
    persons[i].knowsJS);
}                       

    
// Daniel knows JS? undefined
// Armando knows JS? false
// César knows JS? True

    Concept of Classes


    Classes were introduced in JavaScript in ECMAScript 2015. They provide a much simpler and clearer syntax to create objects and deal with inheritance in JavaScript. Classes in JavaScript can be understood as special functions. They serve as blueprints creating objects, referred to as instances of the class. Each class can have one or more instances. 

    class Figura {
    constructor(nombre,base,altura) {
        this.nombre = nombre;
        this.base = base;
        this.altura = altura;
    }
    calcularArea() {
        return this.base * this.altura;
    }
}
class Triangulo extends Figura{
    calcularAreaTriangulo() {
        return super.calcularArea() / 2;
    }
}
let cuadrado = new Figura("cuadrado",4,4);
console.log(cuadrado);
console.log("Área: ",  cuadrado.calcularArea());
let triangulo = new Triangulo("triangulo",4,4);
console.log(triangulo);
console.log("Área: ", triangulo.calcularAreaTriangulo());

    Iterators

    Iterators are objects that access items in a collection (e.g., array) one at a time. And, iterables are objects (or data structures) that define their own iteration behavior, i.e., how each item in a collection is accessed. Built-in objects, such as Array and Map have a default iteration behavior. Using iterators (or iterables), complex container objects can be traversed like lists.

    The for…of loop

    Is new in ECMAScript 6, and it can be used to loop over iterable objects. The for...of loop provides the combined advantages of both the for loop and forEach() method, through break/continue; and concise syntax, respectively.

    Define generators


    Generators are special types of functions that return iterable objects. A function becomes a generator function when it is defined using the function* keyword. Generator functions use the yield operator to pause the function, or send the output and receive input. 

                        
     How generators are used
    
function* idGen() {
       var id = 0;
       while(true)
       yield id++;
}

var gen = idGen(); // calling a generator function

console.log(gen.next().value); // 0
console.log(gen.next().value); // 1


    This code generates a new iterator idGen. Calling a generator function returns a generator object, which can be used to control the generator function.

    In JavaScript, the next() method returns the next item in the sequence. With each iteration, the generator function returns a new value 0, 1, 2 and so on. Every iterator object must maintain its internal state. Generators allow developers to define an iterative algorithm that can maintain its own state through the creation of a single function. You can also try creating your own custom iterator that has its own next() method.

    Yield

    The yield operator is available only inside generator functions and is used to pause and resume generator functions. Its syntax is: 

    
[return_value] = yield [expression];

//expression value returned from the generator

    In the previous example, the yield operator pauses the execution of the idGen() function and returns the new generator value. The execution of the function is resumed the next time next() method (gen.next() method) is called.

    The call() method allows the developer to call a method (defined for an object) from a different object. This method substitutes the this object of the function from the original context to the new object specified in the call() method. 

    var emp1= {
    name:"Daniel",
    id: "01",
    display: function() {
       return this.name + this.id;
       }

var newObj = { name:"NewObject", id:"02"}

console.log(emp1.display.call(newObj)); // NewObject02

    The apply() method is similar to the call() method, except it takes an array as an argument.

    In non-strict mode, if the value of the first argument is null or undefined, it is replaced with the global object. Whereas in strict mode, the first argument takes the value of this in the function being called. 

    //function.apply(thisObj, [objArray]);

    The bind() creates a new function; it is particularly useful when a method will be executed in some other context than when it was originally created in the script. The bind() method returns a copy of the function with the user-specified this value. 

    this.a = 5; // this -> global "window" object
var bnd = {
    a: 10,
    getA: function() { return this.a; }
};

var valA = bnd.getA;

// Create a new function with 'this' bound to bnd

var bndGetA = valA.bind(bnd);

console.log("With bind()",bndGetA()); // 10

// Without the bind statement, bndGetA() will return 5

console.log("Without bind()",valA()); // 5
